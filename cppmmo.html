CppMMO

C++20과 Boost.Asio를 기반으로 구현한 고성능 실시간 멀티플레이어 MMO 서버입니다. 60 TPS 게임 루프, QuadTree 공간 최적화, Server Authority 기반 치트 방지 시스템을 구현했습니다.

프로젝트 개요

CppMMO는 현대적인 C++ 기술을 활용하여 구현한 완전한 멀티플레이어 게임 서버입니다. 실시간 네트워킹, 공간 최적화, 게임 로직 처리 등 MMO 게임에 필요한 모든 핵심 기능을 포함하고 있습니다.

핵심 기능

⚡

60 TPS 게임 루프

매 16.67ms마다 안정적인 월드 업데이트를 수행하는 고성능 게임 루프를 구현했습니다. 정확한 타이밍 제어로 일관된 게임 경험을 제공합니다.

🏗️

Server Authority

서버 권한 기반 게임 시스템으로 클라이언트 치트를 방지합니다. 모든 게임 상태는 서버에서 관리되며, 클라이언트는 입력만 전송합니다.

🌳

QuadTree AOI

공간 분할 알고리즘으로 플레이어 주변 100 유닛 범위 내의 객체만 효율적으로 관리합니다. 대규모 월드에서도 최적의 성능을 보장합니다.

📡

FlatBuffers 통신

구글의 FlatBuffers를 사용한 효율적인 바이너리 직렬화로 네트워크 트래픽을 최소화하고 처리 속도를 극대화했습니다.

🔄

실시간 동기화

60Hz 월드 스냅샷 브로드캐스트로 모든 플레이어의 실시간 위치와 상태를 동기화합니다. 지연 시간을 최소화한 부드러운 멀티플레이어 경험을 제공합니다.

🔐

통합 인증 시스템

ASP.NET Core AuthServer와 Redis 연동으로 안전한 사용자 인증 및 세션 관리를 구현했습니다. 세션 티켓 기반 보안 시스템을 제공합니다.

핵심 패킷 시스템

// 플레이어 입력 처리 (C_PlayerInput)
struct PlayerInput {
    uint8_t inputFlags;     // WASD 비트 플래그 (W=1, S=2, A=4, D=8)
    uint32_t sequenceNumber; // 중복 방지용 시퀀스 번호
    uint64_t clientTime;     // 클라이언트 타임스탬프
};

// 월드 스냅샷 전송 (S_WorldSnapshot)
struct WorldSnapshot {
    uint64_t serverTick;         // 서버 틱 번호
    vector<PlayerState> players;  // 모든 플레이어 상태
    vector<GameEvent> events;     // 게임 이벤트들
};

// 게임 루프 핵심 로직 (60 TPS)
void GameManager::Update() {
    auto startTime = std::chrono::high_resolution_clock::now();
    
    // 1. 대기 중인 명령 처리
    ProcessPendingCommands();
    
    // 2. 게임 월드 업데이트
    UpdatePlayerMovements();
    UpdateQuadTree();
    
    // 3. 월드 스냅샷 전송
    SendWorldSnapshots();
    
    // 4. 다음 틱까지 대기
    WaitForNextTick(startTime);
}

QuadTree 공간 최적화

class QuadTree {
private:
    static constexpr int MAX_OBJECTS = 4;
    static constexpr int MAX_LEVELS = 6;
    
    struct Bounds {
        float x, y, width, height;
    };
    
    int level;
    Bounds bounds;
    vector<shared_ptr<Player>> objects;
    array<unique_ptr<QuadTree>, 4> nodes;

public:
    // AOI 범위 내 플레이어 검색 (O(log n))
    vector<shared_ptr<Player>> GetPlayersInRange(
        const Vec3& position, float range) const {
        
        vector<shared_ptr<Player>> result;
        
        // 범위와 교차하는 노드들 재귀 검색
        if (IntersectsBounds(position, range)) {
            // 현재 노드의 객체들 확인
            for (const auto& player : objects) {
                if (Distance(player->GetPosition(), position) <= range) {
                    result.push_back(player);
                }
            }
            
            // 자식 노드들 재귀 검색
            for (const auto& node : nodes) {
                if (node) {
                    auto childResults = node->GetPlayersInRange(position, range);
                    result.insert(result.end(), 
                                childResults.begin(), childResults.end());
                }
            }
        }
        
        return result;
    }
};

기술적 도전과 해결

🎯 도전: 60 TPS 안정성

문제: 게임 루프의 정확한 타이밍 제어

해결: high_resolution_clock과 sleep_until을 활용한 정밀한 시간 관리로 일정한 16.67ms 간격 유지

⚡ 도전: 성능 최적화

문제: 대규모 월드에서의 연산 부하

해결: 룩업 테이블 기반 방향 계산, QuadTree 공간 분할로 O(log n) 검색 성능 달성

🔒 도전: 메모리 안전성

문제: 멀티스레드 환경에서의 안전한 메모리 관리

해결: smart pointer와 RAII 패턴, 단일 스레드 게임 로직으로 레이스 컨디션 방지

🌐 도전: 네트워크 신뢰성

문제: 패킷 손실과 중복 처리

해결: 시퀀스 번호 기반 중복 방지, 자동 연결 해제 감지 시스템

🔐 도전: 분산 인증 시스템

문제: 게임서버와 AuthServer 간 안전한 인증 연동

해결: Redis 기반 세션 티켓 캐싱으로 빠른 인증 검증, AuthServer API로 캐릭터 정보 동기화
