CppMMO

C++20과 Boost.Asio를 기반으로 구현한 실시간 멀티플레이어 MMO 서버입니다. 개인 PC에서 600명 동시 접속 테스트 완료, 30 TPS 게임 루프, QuadTree AOI 시스템을 구현했습니다.

프로젝트 개요

CppMMO는 C++20과 Boost.Asio를 활용하여 구현한 고성능 멀티플레이어 게임 서버입니다. 개인 PC에서 600명 동시 접속 테스트 완료, 실시간 네트워킹, 성능 최적화 등 MMO 게임 서버의 핵심 기능을 학습하고 구현했습니다.

핵심 기능

⚡

30 TPS 게임 루프

매 16.67ms마다 월드 업데이트를 수행하는 게임 루프를 구현했습니다. high_resolution_clock과 sleep_until을 사용하여 정확한 타이밍 제어를 학습했습니다.

🏗️

Server Authority 아키텍처

서버 중심의 게임 상태 관리 아키텍처를 학습하고 구현했습니다. 서버에서 모든 게임 로직을 처리하고, 클라이언트는 입력 데이터만 전송하는 구조를 구현했습니다.

🌳

QuadTree AOI 시스템

공간 분할 알고리즘을 학습하여 QuadTree를 구현했습니다. 플레이어 주변 100 유닛 범위 내의 객체만 효율적으로 검색하여 O(log n) 시간 복잡도를 달성했습니다.

📡

FlatBuffers 패킷 시스템

Google FlatBuffers를 학습하고 적용하여 효율적인 바이너리 직렬화를 구현했습니다. 스키마 기반 직렬화로 타입 안전성과 성능을 확보했습니다.

🔄

멀티플레이어 동기화

30Hz 월드 스냅샷 브로드캐스트와 패킷 배칭 시스템을 구현했습니다. 배칭을 통해 네트워크 시스템 콜을 60% 감소시켰고, CPU 사용률을 20% 개선했습니다.

🔐

인증 시스템 연동

ASP.NET Core AuthServer와 Redis를 연동하여 사용자 인증 시스템을 구현했습니다. 세션 티켓 기반 인증과 Redis 캐싱을 통해 인증 응답 시간을 50ms에서 5ms로 개선했습니다.

핵심 패킷 시스템

// 플레이어 입력 처리 (C_PlayerInput)
struct PlayerInput {
    uint8_t inputFlags;     // WASD 비트 플래그 (W=1, S=2, A=4, D=8)
    uint32_t sequenceNumber; // 중복 방지용 시퀀스 번호
    uint64_t clientTime;     // 클라이언트 타임스탬프
};

// 월드 스냅샷 전송 (S_WorldSnapshot)
struct WorldSnapshot {
    uint64_t serverTick;         // 서버 틱 번호
    vector<PlayerState> players;  // 모든 플레이어 상태
    vector<GameEvent> events;     // 게임 이벤트들
};

// 게임 루프 핵심 로직 (30 TPS)
void GameManager::Update() {
    auto startTime = std::chrono::high_resolution_clock::now();
    
    // 1. 대기 중인 명령 처리
    ProcessPendingCommands();
    
    // 2. 게임 월드 업데이트
    UpdatePlayerMovements();
    UpdateQuadTree();
    
    // 3. 월드 스냅샷 전송
    SendWorldSnapshots();
    
    // 4. 다음 틱까지 대기
    WaitForNextTick(startTime);
}

QuadTree 공간 최적화

class QuadTree {
private:
    static constexpr int MAX_OBJECTS = 4;
    static constexpr int MAX_LEVELS = 6;
    
    struct Bounds {
        float x, y, width, height;
    };
    
    int level;
    Bounds bounds;
    vector<shared_ptr<Player>> objects;
    array<unique_ptr<QuadTree>, 4> nodes;

public:
    // AOI 범위 내 플레이어 검색 (O(log n))
    vector<shared_ptr<Player>> GetPlayersInRange(
        const Vec3& position, float range) const {
        
        vector<shared_ptr<Player>> result;
        
        // 범위와 교차하는 노드들 재귀 검색
        if (IntersectsBounds(position, range)) {
            // 현재 노드의 객체들 확인
            for (const auto& player : objects) {
                if (Distance(player->GetPosition(), position) <= range) {
                    result.push_back(player);
                }
            }
            
            // 자식 노드들 재귀 검색
            for (const auto& node : nodes) {
                if (node) {
                    auto childResults = node->GetPlayersInRange(position, range);
                    result.insert(result.end(), 
                                childResults.begin(), childResults.end());
                }
            }
        }
        
        return result;
    }
};

기술적 도전과 해결

🎯 도전: 개인 PC에서 600명 동시 접속 테스트

문제: 개인 PC 환경에서 대규모 동시 접속 테스트와 성능 최적화

해결: 분산 스폰 시스템으로 AOI 성능 96% 개선, 메모리 풀링과 배치 처리로 안정적인 대규모 접속 지원

⚡ 도전: 메모리 및 네트워크 최적화

문제: 빈번한 메모리 할당/해제와 네트워크 시스템 콜 오버헤드

해결: 메모리 풀링으로 CPU 사용률 35% 개선, 틱 기반 패킷 배칭으로 네트워크 콜 60% 감소 달성

🔒 도전: 재연결 시스템

문제: 재연결 후 플레이어 입력이 처리되지 않는 버그 발생

해결: 디버깅을 통해 입력 시퀀스 리셋 문제를 발견하고 수정. 세션 추적 메커니즘 추가

🌐 도전: 성능 테스트

문제: 서버 성능과 안정성 검증 방법 학습

해결: Python으로 다중 클라이언트 시뮬레이션 도구 개발. 성능 병목 발견 및 개선 과정 학습
